# 功能描述（Question Illustrations）

## 背景
在 AI Question Gen 的 Step 2 中，教师可选择是否开启 Question Illustrations。开启后，系统会在题目生成完成后，为所选题型生成与题干内容匹配的插图。Step 3 中可以对单题插图进行重新生成。

## 核心目标
- 默认不开启插图生成功能。
- 仅在教师主动开启后，支持选择题型进行插图生成。
- 插图在 Step 3 展示，并支持对单题重新生成。

## 关键交互
- Step 2：
  - 开关默认关闭（即默认所有题不生成插图）。
  - 开启后可选择插图风格与适用题型。
- Step 3：
  - 题目生成完成后批量生成插图。
  - 每题展示插图与“Regenerate Illustration”。
  - 重新生成允许覆盖风格/提示词（按当前配置重新生成）。

## 生成策略（已确认）
- 图像生成：使用真实 AI 图像提供商（已确认）。
- 图像存储与 URL 机制：方案 B，持久化存储到 Supabase Storage。
- API 入参：仅传 prompt（已确认）。

# 实现计划

## 1) 前端交互
- Step 2 保持默认关闭插图开关。
- 开启后，允许选择插图风格与题型；可选风格提示词。
- Step 3 展示插图与重新生成按钮。

## 2) 后端接口
- 新增插图生成接口（批量）：
  - 入参：style、style_prompt、questions[{question_id, prompt, question_type}]
  - 出参：images[{question_id, image_url}]
- 单题重生成复用同一接口（只传单题）。

## 3) 图像生成与存储（已确认）
- 方案 A：不落库，仅返回可访问 URL（或 data URL），前端显示即可。
- 方案 B：生成后持久化（对象存储 + DB 存档），返回可复用 URL（Supabase Storage）。

## 4) 前后端联调
- Step 3：生成题目后批量请求插图并渲染。
- 单题重生成：调用接口刷新该题图片。
- 异常与 loading 状态处理。

# 关于“只返回 URL”的机制说明（备选方案）

## 什么是“只返回 URL，不落库”
- 后端调用图像模型生成图片后，直接返回一个可访问的 URL（或临时 data URL）。
- 不保存到数据库，也不永久存储到对象存储。
- 页面刷新或会话结束后，需要重新生成或重新请求。

## 优点
- 实现快、成本低。
- 不涉及存储、权限、清理策略。

## 缺点
- 图片不可复用；用户重新进入页面可能失效。
- 无法做历史追溯或内容审计。
